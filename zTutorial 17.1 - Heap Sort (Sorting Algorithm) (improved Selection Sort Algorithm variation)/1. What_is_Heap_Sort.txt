Selection Sort is a sorting algorithm that sorts a list by repeatedly selecting the smallest (or largest) 
element from the unsorted subarray of the initial unsorted List and moving it to the sorted subarray of the 
initial unsorted List. 


What is Heap Sort?

Heap Sort is mainly an optimised variation of Selection Sort. 

Selection Sort repeatedly finds the smallest element from the unsorted part of the list and places it into 
its correct sorted position. The problem is that finding that smallest element each time requires scanning 
the unsorted part linearly, which takes O(n) time for each pass. Heap Sort improves this by using the 
Heap Data Structure to find and remove the largest or smallest element more efficiently, in O(log n).


//////////////////////////////////////////////


Is the Heap Sort Algorithm implemented iteratively or recursively?

- The overall Heap Sort Algorithm is more commonly implemented iteratively
- The 'max_heapify()' function or 'max_heapify()' function is more commonly implemented recursively
  (implementing it iteratively is possible too but recursive implementation is easier to understand 
  and less complicated to implement)

In this tutorial we will be learning how to implement the overall Heap Sort Algorithm iteratively,
and the 'max_heapify()' function or 'max_heapify()' function recursively.

How does Heap Sort Algorithm work?

Here are the steps of Heap Sort Algorithm:
Using Max Heap Data Structure:
1. Build a Max Heap Data Structure from the unsorted list with Max Heapify operation
2. Since the largest element is always at the root, swap the largest element/root with the last element 
   in the Max Heap Data Structure, and remove the largest element (now the last element)
3. Reduce the size of the Max Heap Data Structure by 1 (since we removed the largest element (now the 
   last element))
4. Max Heapify operation the root again, since it is the only node that does not obey the Max Heap
   property to restore the Max Heap property 

Repeat steps 1 to 4 until the size of the Max Heap Data Structure becomes 1, when the array will 
then be sorted in ascending order

Explaining the Big O Notation of Time and Space Complexity of Heap Sort:

For Big O Notation of Time Complexity of Heap Sort:
Recall the steps of Heap Sort Algorithm from the 'Here are the steps of Heap Sort Algorithm' section,
1. Heap-building stage (aka step 1)
   Build a Heap Data Structure from the initial unsorted List.
   - If we are using a Max Heap Data Structure for ascending order, we will rearrange the elements such that 
   every parent node is greater than or equal to its children
   - If we are using a Min Heap Data Structure for descending order, we will rearrange the elements such that 
   every parent node is smaller than or equal to its children

   This heap-building stage takes O(n) time. Even though heapify by itself takes O(logn) time, when building 
   the heap bottom-up from all non-leaf nodes, the total time is actually O(n), not O(nlogn).

2. Repeated extraction + heapify stage (aka steps 2 to 4)
   Now we repeatedly do the following:
   - take the root element, which is the maximum element in a Max Heap Data Structure or the minimum element 
     in a Min Heap Data Structure
   - swap it with the last element in the unsorted section
   - reduce the size of the unsorted section / heap by 1
   - restore the heap property by heapifying the root again

   We repeat this until the heap size becomes 1.

   Since we do this roughly n times, and each heapify operation takes O(logn) time in the worst case, this stage 
   takes:
   O(n) × O(logn) = O(n logn)

   This is because during each heapify operation, the element at the root may have to travel down the Heap Data 
   Structure from the root to a leaf node, and the height of a Heap Data Structure is logn.


For Big O Notation of Space Complexity of Heap Sort:
Heap Sort Algorithm is usually considered an in-place sorting algorithm. This is because:
- the Heap Data Structure is stored inside the same array as the input list
- no extra temporary array of size n is needed

So the extra memory required apart from the original array is very small and is the Big O Notation of Space 
Complexity of Heap Sort is hence O(1).

          Selection Sort     |        O(1)	  	    |       O(n^2)
            Heap Sort        |        O(1)	  	    |     O(n log n)

with a Big O Notation of Time Complexity of O(n^2).

Advantages and Disadvantages of Heap Sort Algorithm compared to other sorting algorithms:

Advantages of Heap Sort:
1. Efficiency: Heap Sort has a time complexity of O(n log n) for both average and worst-case scenarios, 
   making it efficient for sorting large datasets. Its use of the heap data structure ensures that each 
   insertion and extraction operation is performed in logarithmic time.
2. In-Place Sorting: Heap Sort is an in-place sorting algorithm, meaning it sorts the data within the 
   original array without requiring additional memory beyond a constant amount. This can be advantageous 
   when working with large datasets where minimizing memory usage is crucial.
3. No Worst-Case Performance Variability: Unlike Quick Sort, Heap Sort's performance does not degrade 
   significantly in worst-case scenarios. Its time complexity remains O(n log n) regardless of the input 
   data's characteristics.
4. Predictable Performance: Heap Sort provides predictable performance, as its time complexity does not 
   vary with different types of input, unlike algorithms with variable performance like Quick Sort.
5. Suitable for Limited Memory Environments: Since Heap Sort does not require additional memory beyond a 
   constant amount, it is well-suited for environments with limited memory.

Disadvantages of Heap Sort:
1. Not Stable: Heap Sort is not a stable sorting algorithm, meaning that it does not necessarily maintain 
   the relative order of equal elements. This can be a disadvantage in applications where the stability 
   of the sorting process is important.
2. Slower for Some Cases: Although Heap Sort has a time complexity of O(n log n), in practice it can be 
   slower than other O(n log n) algorithms like Merge Sort due to higher constant factors and less 
   cache-friendly memory access patterns.
3. Complex Implementation: Implementing Heap Sort can be more complex compared to simpler sorting algorithms l
   ike Bubble Sort or Insertion Sort. The process involves managing the heap property and performing 
   operations like heapify, which can add to the complexity.
4. Less Intuitive: The heap data structure and the process of maintaining the heap property during sorting 
   can be less intuitive and harder to understand compared to simpler sorting algorithms.
5. Not Adaptive: Heap Sort does not take advantage of existing order in the data. Unlike adaptive sorting 
   algorithms that can perform better on nearly sorted data, Heap Sort performs consistently regardless 
   of the initial ordering of the data.

Why is Heap Sort called 'Heap Sort'?

Heap sort is called as such because it is a sorting algorithm that sorts a list based on the Heap 
Data Structure.
