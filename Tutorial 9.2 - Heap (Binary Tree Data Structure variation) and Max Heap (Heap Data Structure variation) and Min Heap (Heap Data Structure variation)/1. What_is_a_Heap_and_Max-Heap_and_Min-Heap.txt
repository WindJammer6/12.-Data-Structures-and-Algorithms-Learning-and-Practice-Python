Whats a Heap?
A Heap is a further variation of the Binary Tree Data Structure. The difference/modification between a Binary 
Tree and Heap is that a normal Binary Tree is unordered, while a Heap is partially ordered (by some manner) 
according to the Heap Property.

Similar to Binary Tree, Heap can handle duplicates (unlike Binary Search Trees, which cannot handle duplicates). 
Usually, the Heap Data Structure is implemented as one of its further variations, more commonly in the form of 
a Max-Heap or Min-Heap, depending of the type of Heap Property used.

Heap must also be a complete binary tree (see what this means in the file '1. What_is_a_Binary_Tree_and_Binary_
Search_Tree.txt', in the folder 'Tutorial 9.1 - Binary Trees (Tree Data Structure variation) and Binary Search 
Trees (Binary Tree Data Structure variation)', in the section 'State types of Binary Trees and their names/terms 
(applies to all its variations)'), due to its specific requirements for efficient operations (We will be able 
to understand this once we learn how the Heap's partial ordering/operations works)


What is the Heap Property?
The Heap Property is a key characterstic of Heap Data Structure that defines the relationship between the parent
nodes and child nodes, which gives Heap Data Structure its partially ordered characteristic. It comes in 2 types, 
- Max-Heap Property: which states that the parent node must always be greater or equal to its children nodes, and
                     hence the root node must be storing the largest value
- Min-Heap Property: which states that the parent node must always be smaller or equal to its children nodes, and
                     hence the root node must be storing the smallest value



Whats a Max-Heap?
A Max-Heap is a further variation of the Heap Data Structure, which is essentially a Heap Data Structure being
partially ordered according to the Max-Heap Property. 


Whats a Min-Heap?
A Min-Heap is a further variation of the Heap Data Structure, which is essentially a Heap Data Structure being
partially ordered according to the Min-Heap Property. 



By the way, Heap are called as such as due to their partially ordered characteristic, via the Heap Property 
(Max-Heap Property or Min Heap Property), causing the nodes to be 'heaped'/'piled' together following the
corresponding Heap Property. If the Heap is following the Max-Heap Property, the largest node is always at 
the top of the pile followed by the moderately large nodes, with the smallest nodes at the bottom of the pile.
If the Heap is following the Min-Heap Property, the smallest node is always at the top of the pile followed 
by the moderately small nodes, with the largest nodes at the bottom of the pile.

How is a Max-Heap partially ordered by the Max-Heap Property and Min-Heap partially ordered by the Min-Heap Property?
The Max-Heap and Min-Heap ordering depends on what data type it is storing in its Nodes. Usually it will store numbers 
(ints/float data types). If it is storing other data types then there will be a completely different way to 
order/organise our Max-Heap and Min-Heap.

In this tutorial, we will only talk about if the Max-Heap and Min-Heap is storing numbers.



So how is a usual Max-Heap (storing numbers) ordered? 
In a Max-Heap, the nodes are organized according to the Max-Heap Property, which ensures that each parent node is greater 
than or equal to its children. This means that the largest node in the Max-Heap is always the root node.

For a particular Node in a Max-Heap, the left and right sub-trees are organised to be smaller than the number of the 
particular Node chosen.

Visual representation on how a Max-Heap could look like:
          [40]
          /  \ 
      [19]    [15]
      /  \    /  \
   [12]  [7][15] [12]
   /   
 [6]            

Lets use the example above. Taking the root Node, storing 40 as our particular Node chosen, all the Nodes at the
left and right sub-trees of our particular Node (40) are storing numbers smaller than 40.

This applies to every Node in the Max-Heap Data Structure. Taking the Node storing 19 as our particular Node chosen, 
all the Nodes at the left and right sub-trees of our particular Node (19) are storing numbers smaller than 19 (12, 7 and 6).

Another thing to note is that there can be duplicates, where 2 Nodes are storing the same number. 



So how is a usual Min-Heap (storing numbers) ordered? 
In a Min-Heap, the nodes are organized according to the Min-Heap Property, which ensures that each parent node is smaller 
than or equal to its children. This means that the smallest node in the Min-Heap is always the root node.

For a particular Node in a Min-Heap, the left and right sub-trees are organised to be larger than the number of the 
particular Node chosen.

Visual representation on how a Min-Heap could look like:
           [7]
           /  \
       [12]    [14]
       /  \    /  \
    [15]  [14][20] [23]
    /
 [88]
Lets use the example above. Taking the root Node, storing 7 as our particular Node chosen, all the Nodes at the
left and right sub-trees of our particular Node (7) are storing numbers larger than 7.

This applies to every Node in the Min-Heap Data Structure. Taking the Node storing 19 as our particular Node chosen, 
all the Nodes at the left and right sub-trees of our particular Node (12) are storing numbers larger than 12 (15, 14 and 88).

Another thing to note is that there can be duplicates, where 2 Nodes are storing the same number. 



Note: The same terminology is used here for Min-Heap and Min-Heap as for in (General) Tree Data Structures
(such as Root Node, Leaf Node, Parent Node, Levels, etc.) (refer to '1. What_is_a_Tree_and_General_Tree.txt'
in 'Tutorial 9 - Trees and General Trees (Data Structure)')

How can we convert a Complete Binary Tree Data Structure to a Min-Heap Data Structure or Max-Heap Data 
Structure?
In order to convert a Binary Tree Data Structure to a Max-Heap Data Structure/Min-Heap Data Structure, we 
need to rearrange the Nodes in the Binary Tree Data Structure such that they fulfill the Max-Heap 
Property/Min-Heap Property, which is when they will become a Max-Heap Data Structure/Min-Heap Data Structure.

(Note that if the Binary Tree Data Structure is not complete, then we cannot convert it to a Heap Data 
Structure because a Heap Data Structure must be a complete binary tree)


The way to rearrange the Nodes in the Binary Tree Data Structure such that they fulfill the Max-Heap Property
is via a process called Max-Heapify, while the way to rearrange the Nodes in the Binary Tree Data Structure 
such that they fulfill the Min-Heap Property is via a process called Min-Heapify.

What is Max-Heapify?
The Max-Heapify operation is used to convert a regular Binary Tree Data Structure to the Max-Heap Data Structure.

Here are the steps of Max-Heapify:
1. During the first recursive loop,

   Since Binary Tree Data Structures are represented using arrays, we start from the Node/element at index 
   'midpoint - 1', which is the last non-leaf Node in the array representation of the complete Binary Tree 
   Data Structure. This Node is the rightmost Node that is guaranteed to still have at least one child, 
   while all Nodes after it are leaf Nodes in the last level of the tree. Therefore, heapification begins 
   from this Node and then works upwards toward the Root Node (see the 'notes' section for proof).

2. Select the larger child Node out of the 2 child Nodes of the current (parent) Node.

3. Since in a Binary Tree Data Strucutre, the left child Node and right child Node are not ordered (which 
   means there is no order between them, e.g. like for BST Data Structure, the left child Node and right
   child Node are ordered in such a way that the left child Node must be smaller than the right child
   Node), we will need to check if the larger child Node out of the 2 child Nodes is larger than the 
   current (parent) Node.
   
   a. If the larger child Node out of the 2 child Nodes of the current (parent) Node is larger than the
      current (parent) Node, swap the larger child Node out of the 2 child Nodes of the current (parent) 
      Node with the current (parent) Node.

      Since at the position of the current (parent) Node, it is now storing the swapped larger child 
      Node out of the 2 child Nodes, and the position of the larger child Node out of the 2 child 
      Nodes is now the smaller current (parent) Node, move from the current (parent) index (which
      is now storing the swapped larger child Node out of the 2 child Nodes) to the larger child 
      Node out of the 2 child Nodes (which is now storing the smaller current (parent) Node). Then
      recursively repeat more loops of step 1, 2 and 3, which is will result in the current (parent)
      index to seemingly travel down the Binary Tree Data Structure.

   b. Otherwise, if the larger child Node out of the 2 child Nodes of the current (parent) Node is not 
      larger than the current (parent) Node or if the current (parent) Node has no child Nodes (the
      current (parent) Node has reached a leaf Node), there is no need to swap the larger child Node 
      out of the 2 child Nodes of the current (parent) Node with the current (parent) Node, this
      sub-tree of the Binary Tree Data Structure can be said to satisfy the Max-Heap Property and we 
      can end the Max-Heapify operation. (since starting is only leaf node as the subtrees which technically 
      is already fulfilling the max-heap property and while working your way across to the left and 
      up, the subtrees will all already be fulfill max-heap property done by previous recursive iteration 
      hence you can assume this)



Note: 
- This Max-heapify operation is the repeated, but starting from 'midpoint - 1' and then all the way to 
  the root node (index 0). You can kinda visulise this process as multiple traversals down the Binary 
  Tree Data Structure for every Node as the starting current (parent) Node starting from 
  'midpoint - 1' to first index (aka Root Node).

  This step is not part of max-heapify process, but is more for building the Heap Data Structure 
  from the (complete) Binary Tree Data Structure.

- Why do we start from 'midpoint - 1'? And why the Node/element at the most right of the second last 
  level of the Binary Tree Data Structure, right before the last level of the Binary Tree Data Structure 
  (where all the leaf Nodes are at) is guranteed to be the element at the 'midpoint - 1' of the array?
  When a complete Binary Tree Data Structure is stored as an array, the Nodes are placed into the array
  level by level, from left to right. Because of this arrangement, all Nodes in the last level of the 
  tree — which are all leaf Nodes — appear in the later half of the array. Since leaf Nodes do not 
  have any children, they already trivially satisfy the Heap Property and do not need to be heapified.

  Therefore, the first Node that may need heapification is the last non-leaf Node, i.e. the rightmost 
  Node in the second-last level of the Binary Tree Data Structure. In the array representation of a 
  complete Binary Tree, this Node is guaranteed to be at index 'midpoint - 1' (equivalently, 'n // 2 - 1'), 
  which is why we begin there.

  From that starting point, we work backwards through the array, from index 'midpoint - 1' all the way 
  to index 0 (the Root Node), heapifying each Node/sub-tree along the way.

  This works because heapification is done from the bottom upwards. By first heapifying the smaller 
  sub-trees near the bottom of the tree, we ensure that when we move up to a parent Node, its child 
  sub-trees already satisfy the Heap Property. This allows heap construction to be viewed as solving 
  many smaller subproblems first, then combining them to solve the full problem at the Root Node.

The Min-Heapify operation is used to convert a regular Binary Tree Data Structure to the Min-Heap Data Structure.

Here are the steps of Min-Heapify:
1. During the first recursive loop,

   Since Binary Tree Data Structures are represented using arrays, we start from the Node/element at index

   means there is no order between them, e.g. like for BST Data Structure, the left child Node and right
   child Node are ordered in such a way that the left child Node must be smaller than the right child
   Node), we will need to check if the smaller child Node out of the 2 child Nodes is smaller than the
   current (parent) Node.

   a. If the smaller child Node out of the 2 child Nodes of the current (parent) Node is smaller than the
      current (parent) Node, swap the smaller child Node out of the 2 child Nodes of the current (parent)
      Node with the current (parent) Node.

      i. Since at the position of the current (parent) Node, it is now storing the swapped smaller child
         Node out of the 2 child Nodes, and the position of the smaller child Node out of the 2 child
         Nodes is now the larger current (parent) Node, move from the current (parent) index (which
         is now storing the swapped smaller child Node out of the 2 child Nodes) to the smaller child

Explaining how Deletion operation works in Max-Heap Data Structure and Min-Heap Data Structure:
Deletion for Heap Data Structure is NOT the same as that for the BST Data Structure. In a Heap Data Structure, 
we usually delete the root node, because that is the most useful element:
- in a Max-Heap Data Structure, the root node is the largest
- in a Min-Heap Data Structure, the root node is the smallest

Here are the steps of deletion operation of root node for Max-Heap Data Structure:
1. Store the root value if needed.
2. Replace the root with the last element in the heap.
3. Remove the last element from the heap.
4. Perform Max-Heapify starting from the root. This pushes the new root downward until the Max-Heap 
   Property is restored.

Explaining how Insertion operation works in Max-Heap Data Structure and Min-Heap Data Structure (This 
process is called heapify-up or bubble-up):
Here are the steps of insertion operation for Max-Heap Data Structure:
1. Insert the new value at the next available position to maintain completeness.
2. Compare it with its parent. 
3a. If it is larger than its parent, swap upward. Repeat steps 1 and 2.
 b. Else, the Max-Heap Property is restored.
